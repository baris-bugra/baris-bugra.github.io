Introduction

The main purpose of the project is to develop a system that is able to detect, track and count people in an indoor area. These problems are included under the computer vision field. Computer vision basically focuses on extracting meaningful information from digital images or videos. In our system, people in the input video can be recognized and tracked, the number of people can be counted and a heat map can be created.

Used Technologies

















Detection

Detection is one of the main part of the project. We used YOLOv5 as the detection algorithm. YOLOv5 contains a family of detection architectures and pre-trained models. We did not train a model of our own. Instead, we used a model which is pretrained on the MS COCO dataset. This dataset contains images of different kinds of objects including humans. However, the weights in the model that we used improves the performance and accuracy when detecting persons.
We use the algorithms with the parameter “—classes 0” to detect only - also count and track- humans in the videos. Detection quality depends on the pre-trained model given to the algorithm as well as the used video’s quality and the angles that humans appear in the camera.


Counting

The goal of object counting task is to count the number of object instances in a single image or video sequence. In our case, the objects are humans. There are different methods for object counting in computer vision. We make use of detections made by YOLOV5.
When detection is done it can be assumed that counting is very simple processes. However, without high accuracy detection or tracking, counting is a tough task. It can produce nonsense numbers.
Counting’s accuracy is very related with the detection, tracking and re-identification algorithms’ accuracy. When we can track and re-identificate people with high accuracy, there is nothing wrong on the counting stage.
We are using 2 counters, one of them is to keep exact detected people number in the current frame and the other one is to keep total number of people in the whole video.

To improve counting

Track better: Track people among the frames and do not give new ids to existing person in the video.
Re-identificate person: When a person couldn’t be detected for a couple of frames, re identificate this person when it appears in the video again.


Tracking and Re-identification

Object tracking is the task of taking an initial set of object detections, and creating a unique ID for each of the detected object and associate these detections across frames in the video. Since, oftenly, there are more than one person in a frame, our problem is a multiple object tracking problem. For solving this problem, the DeepSORT algorithm is used. DeepSORT is a recent tracking algorithm that is an improvement to the SORT(Simple Online and Realtime Tracking) method by using deep learning metrics.
Tracking can be affected by; Occlusion by another object and Leaving the scene for some time It may cause identity switches or incorrect assignment of ids. To solve this effect we have tested different re-identification models ( MobileNetV2, ResNet, OSNet etc.)


Heat Map

In the most general sense, heatmap is a representation of data in the form of a map or diagram in which data values are represented as colors.
Basically, we found out how many bounding boxes are placed on each pixel in all frames of the video and colored all pixels with different colors according to that number. For example, the highest number of occurrences colored with red color, smallest number of occurrences colored with blue etc.
Heatmaps help us identify the areas where the people density is high.
